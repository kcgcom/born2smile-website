# category-hubs 운영 가이드

블로그 카테고리 허브(`/blog/[category]`)의 큐레이션 설정을 관리하는 내부 전용 디렉토리입니다.
사용자에게 직접 공개되는 URL이 아니라, 허브 페이지를 렌더링하기 위해 서버/빌드 시점에 읽는 설정 코드입니다.

## 구조

- `shared.ts` — 공통 타입/헬퍼
- `{category}.ts` — 카테고리별 허브 설정
- `index.ts` — 전체 허브 집계 + 자동 검증

예시:
- `implant.ts`
- `prevention.ts`
- `health-tips.ts`

## 수정 원칙

### 1) 자기 카테고리 글만 연결
각 허브 파일에는 **해당 카테고리의 글 slug만** 넣습니다.

예:
- `implant.ts`에는 `implant` 카테고리 글만 사용
- `prevention.ts`에는 `prevention` 카테고리 글만 사용

다른 카테고리 글을 넣으면 `index.ts`의 검증에서 빌드가 실패합니다.

### 2) slug는 실제 발행 메타 기준으로 작성
허브는 `BLOG_POSTS_META`를 기준으로 검증됩니다.
즉, slug를 추가할 때는 반드시 실제 글 메타에 존재해야 합니다.

추천 확인 방법:
- `lib/blog/generated/posts-meta.ts` 검색
- 또는 관리자에서 발행된 slug 확인

### 3) 질문/추천/섹션은 역할을 나눠서 구성
허브는 단순 목록이 아니라 읽는 순서를 안내해야 합니다.

권장 역할:
- `questions` — 가장 자주 찾는 질문 진입점
- `featuredSlugs` — 가장 먼저 읽을 글
- `sections` — 상황별/단계별 읽기 묶음

### 4) FAQ는 글을 반복 요약하지 말고 의사결정 질문 중심으로 작성
좋은 FAQ 예시:
- 누구에게 해당되는가?
- 언제 검진/치료가 필요한가?
- 어떤 기준으로 선택하는가?
- 집에서 무엇을 관리해야 하는가?

## 권장 구성 기준

### questions
- 권장 3개
- 질문 문장형으로 작성
- 초진/상담 전 사용자가 실제로 검색할 법한 표현 사용

### featuredSlugs
- 권장 3개
- 입문 1개 + 비교 1개 + 관리 1개 조합 권장

### sections
- 권장 3개
- 각 섹션당 2~4개 글 권장
- 섹션 제목은 사용자의 상황 기준으로 작성
  - 예: `치료 전 판단`, `시술과 회복`, `오래 쓰는 관리와 비교`

### faq
- 권장 3개 이상
- 답변은 짧고 명확하게 작성
- 과도한 의료 단정 표현은 피하고, 상담/검진 필요성을 자연스럽게 안내

## 수정 절차

1. 해당 카테고리 파일을 수정
2. slug가 실제 존재하는지 확인
3. `pnpm lint`
4. `pnpm build`
5. 가능하면 배포 후 허브 URL 스모크 체크

## 자동 검증

`index.ts`에서 아래를 자동 검증합니다.

- 허브에 적은 slug가 실제 존재하는지
- slug가 해당 카테고리에 속하는지

즉 아래와 같은 실수를 자동으로 잡습니다.
- 오타 slug 입력
- 다른 카테고리 글 혼입
- 삭제된 글을 계속 참조

## 운영 팁

- 새 글을 발행한 뒤 허브에 바로 반영할 가치가 있는지 먼저 판단합니다.
- 모든 글을 허브에 억지로 넣지 말고, 대표 글만 연결합니다.
- 허브는 “많이 보여주는 페이지”보다 “잘 안내하는 페이지”가 더 좋습니다.
- 카테고리별 허브 톤은 일관되게 유지하되, 질문/섹션은 카테고리 특성에 맞게 차별화합니다.

## 주의

- 이 디렉토리는 **내부 설정 코드**이며 공개 URL이 아닙니다.
- URL은 `app/blog/[category]/page.tsx`가 담당합니다.
- 허브 구조를 바꾸려면 설정 파일뿐 아니라 페이지 렌더링 로직도 함께 확인해야 합니다.
