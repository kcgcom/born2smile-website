# 홈페이지 & 블로그 개선 아이디어

> 2026-02-15 브레인스토밍 결과 정리. 우선순위별로 분류.

## 홈페이지

### 1. 환자 후기/신뢰 섹션 추가
- 의료진 소개와 시설 안내 사이에 "환자분들의 이야기" 섹션 추가
- 텍스트 기반 후기 카드 (별점 + 한 줄 요약 + 진료 과목 태그)
- 신뢰도 향상 효과

### 2. 블로그 하이라이트 섹션
- CTA 배너 위에 "최신 구강건강 정보" 섹션 추가
- 최근 블로그 3~4개를 카드로 노출
- 홈페이지 → 블로그 유입 경로 확보, 체류 시간 개선

### 3. 진료 과목 카드 그리드
- 6개 진료 과목을 아이콘 카드 그리드로 홈페이지에 요약 표시
- 각 카드 클릭 → 해당 진료 상세 페이지로 이동
- `lib/constants.ts`의 `TREATMENTS` 데이터 활용

### 4. 자주 묻는 질문 아코디언
- `lib/treatments.ts`의 FAQ 데이터에서 공통 질문 5~6개 선별
- 아코디언 UI로 홈페이지에 노출
- FAQPage JSON-LD 스키마 추가 → SEO 개선

### 5. Hero 섹션 동적 요소
- 시설 사진 배경 슬라이드쇼 또는
- 핵심 강점 순환 타이핑 애니메이션 ("서울대 출신 전문의", "꼼꼼한 설명" 등)

## 블로그

### 6. 검색 기능 ✅ 구현됨
- `BLOG_POSTS_META`의 title, subtitle, excerpt 대상 클라이언트 사이드 텍스트 검색
- 카테고리/태그 필터와 연동

### 7. 인기글/추천글 표시
- Firestore `blog-likes/{slug}`의 좋아요 수 기준 "인기글" 배지
- 블로그 상단 인기글 캐러셀

### 8. 카테고리별 랜딩 페이지
- `/blog/category/임플란트` 같은 카테고리 전용 URL
- 카테고리별 고유 메타데이터와 JSON-LD
- 진료 상세 페이지에서 관련 블로그 카테고리로 직접 링크

### 9. 목차(TOC) 자동 생성
- `post.content`의 heading 배열 활용
- 데스크톱: sticky 사이드바, 모바일: 상단 드롭다운

### 10. 읽기 시간 자동 계산
- `generate-blog-meta.ts`에서 content 글자 수 기반 자동 계산 (한국어 기준 분당 ~500자)
- 수동 `readTime` 입력 불필요

### 11. 관련 포스트 알고리즘 개선
- 현재: 같은 카테고리만 추천
- 개선: 태그 기반 매칭 추가 (카테고리가 달라도 관련 태그 공유 시 추천)

### 12. 블로그 포스트 콘텐츠 리치화
- `BlogPostSection`에 리스트, 강조 박스, 비교 테이블 등 추가 타입 지원
- `list?: string[]`, `callout?: { type, text }`, `table?: { headers, rows }`
