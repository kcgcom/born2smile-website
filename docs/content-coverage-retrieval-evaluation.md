# 콘텐츠 커버리지 검색 평가

## 목적

콘텐츠 커버리지 엔진이 주제와 관련된 근거 문단을 안정적으로 찾는지 평가하기 위한 기준선 문서다. 관리자 화면 `/admin/content/strategy/retrieval-review`에서 고정된 후보 스냅샷을 검토하며, 운영 라벨은 Supabase `api_cache`에 저장한다. 완료된 50건은 `lib/content-coverage/generated/retrieval-review-baseline.json`에 개인정보 없이 고정해 평가 기준으로 사용한다.

현재 후보 스냅샷은 `gemini-embedding-2`, 768차원 임베딩으로 만든 5개 대표 주제별 상위 10건, 총 50건이다. 후보 생성 시점 이후 검색 로직을 바꾸더라도 이 스냅샷은 최초 기준선 비교를 마칠 때까지 유지한다.

## 판정 단위와 원칙

판정 단위는 콘텐츠 전체가 아니라 하나의 근거 문단(`EvidenceUnit`)이다. 제목과 소제목 경로는 문맥으로 사용할 수 있지만, 후보에 포함되지 않은 다른 문단의 설명을 현재 문단의 근거로 합산하지 않는다.

| 라벨 | 판정 기준 |
|---|---|
| `relevant` | 문단이 주제의 필수 개념 하나 이상을 직접적이고 실질적으로 설명한다. |
| `partial` | 일부 기준의 보조 근거가 되지만 적용 범위가 좁거나, 단독으로 사용하기에는 설명이 부족하다. |
| `irrelevant` | 표현만 비슷하거나 진료·환자·치아 위치·상황이 달라 해당 주제의 근거로 사용할 수 없다. |

정확한 주제 단어의 포함은 필수 조건이 아니다. 대신 문단 자체가 평가하려는 관계나 사실을 뒷받침해야 한다. 예를 들어 “재료와 뼈이식 여부에 따라 환자 부담액이 달라진다”는 문장은 `비용`이라는 단어가 없어도 비용 결정 요인의 근거가 된다. 반면 소재 종류만 나열한 문단을 보고 가격 차이를 추론해서는 안 된다.

한 문단이 주제 전체를 설명할 필요도 없다. 라벨은 검색 근거로서의 적합성을 뜻하며, 주제 전체의 충족 여부는 이후 개념별 근거를 합산해 별도로 판단한다.

## 1차 기준선 현황

2026-07-14 현재 5개 주제 50건의 사전검토가 완료됐다. `codex-prereview`는 최종 임상 검수가 아니라 검색 품질 평가를 위한 사전 라벨이다.

| 주제 | 관련 | 부분 관련 | 무관 | 상위 3개 유효 비율 | 상위 10개 무관 비율 |
|---|---:|---:|---:|---:|---:|
| 교정 통증과 부작용 | 3 | 4 | 3 | 100% | 30% |
| 치과 비용과 보험 | 3 | 0 | 7 | 66.7% | 70% |
| 앞니 치료 | 3 | 4 | 3 | 100% | 30% |
| 소아 치아 외상 | 3 | 2 | 5 | 100% | 50% |
| 구강 위생 | 5 | 5 | 0 | 100% | 0% |
| 합계 | 17 | 15 | 18 | 93.3% | 36% |

평가 목표는 주제별 상위 3개 유효 비율 80% 이상, 상위 10개 무관 비율 20% 이하다. 구강 위생만 두 목표를 모두 충족한다. 나머지 네 주제는 무관 비율 목표를 충족하지 못하며, `치과 비용과 보험`은 두 지표 모두 미달한다.

## 주제별 발견 사항

### 교정 통증과 부작용

- 정상적인 초기 통증과 투명교정 장치 자극 근거는 상위권에서 잘 검색됐다.
- 크라운과 신경치료의 “정상 통증 대 위험 신호” 문단이 5~7위에 연속으로 포함됐다.
- 임베딩이 진료 종류보다 통증의 시간 경과와 위험 신호라는 서술 구조를 강하게 반영했다.
- 식사·위생·장치 탈락 문단은 관련은 있지만 명세의 치근 흡수, 잇몸 퇴축, 위험 신호를 직접 설명하지 않아 보조 근거에 그쳤다.

### 치과 비용과 보험

- 실제 관련 근거는 임플란트 보험·견적, 교정과 임플란트 병행 비용, 소아 실란트 보험 문단 3건이었다.
- 치과 선택, 일반 상담, 치료 선택, 소재 선택, 예산의 단순 언급이 비용·보험 근거로 과도하게 검색됐다.
- 상위 10개에 실손·민간보험 설명이 없었고, 일반적인 치료비 구성이나 추가 비용 확인 방법도 부족했다.
- 정확한 `비용`·`보험` 단어를 강제할 필요는 없지만, 비용 차이 또는 보험 적용 관계가 문단에서 직접 성립해야 한다.

### 앞니 치료

- 보철 선택지, 라미네이트 적응증, 앞니 크라운의 심미·교합 기준은 유효한 근거였다.
- 명시적 제외 대상인 앞니 부분교정이 두 건 포함됐다.
- 일반 치아 파절, 소아 외상, 일반 임플란트·브릿지 문단은 일부 개념을 지원하지만 앞니 고유의 심미성·잇몸선·교합 조건이 없어 부분 관련에 그쳤다.
- 어금니 레진·인레이 비교가 치료 선택이라는 의미적 유사성으로 포함됐다.
- 레진·라미네이트·크라운 비교, 앞니 결손의 브릿지·임플란트 기준, 치료 기간·회복·비용 근거가 부족했다.

### 소아 치아 외상

- 파절 조각 보관, 지혈, 유치·영구치 구분, 빠른 내원 신호를 설명하는 직접 근거가 2~4위에 모였다.
- 1위 요약문은 주제 범위는 정확하지만 실제 응급 행동을 설명하지 않아 부분 관련에 그쳤다.
- 5위는 앞니를 부딪친 아이의 조기 확인 필요성만 설명해 보조 근거로 제한됐다.
- 6위 이후에는 충치·발치 후 관리, 부정교합, 첫 검진 등 외상과 다른 일반 소아 콘텐츠가 연속으로 포함됐다.
- 임베딩이 `아이`, `영구치`, `치료 후 관리`, `조기 확인` 같은 공통 표현을 외상 문맥보다 강하게 반영했다.
- 유치와 영구치의 응급 대처 및 내원 기준은 사전 라벨과 별도로 의료진 최종 확인이 필요하다.

### 구강 위생

- 상위 10개에 무관 후보가 없었고, 평가 대상 중 검색 품질이 가장 안정적이었다.
- 치실·치간칫솔·워터픽의 대상별 선택, 교정 상황별 관리, 칫솔질 빈도와 방법, 정기검진·스케일링 근거가 고르게 검색됐다.
- 글의 범위만 소개하는 요약문과 실제 사용법이 없는 도입부는 부분 관련으로 제한됐다.
- 칫솔 보관과 교체는 유용한 인접 영역이지만 현재 명세의 세부 기준을 직접 충족하지 않아 보조 근거로 분류됐다.
- 구강 위생처럼 개념과 표현이 비교적 구체적인 주제에서는 임베딩 검색만으로도 높은 관련성을 확보할 가능성을 확인했다.

## 공통 오류 유형

현재 사유 태그에서 반복적으로 확인되는 오류는 다음과 같다.

1. **인접 주제 과대평가**: 상담, 선택, 관리처럼 넓은 의미가 핵심 개념보다 우선한다.
2. **적용 범위 불일치**: 다른 진료, 다른 환자군, 다른 치아 위치의 설명을 그대로 전이한다.
3. **단순 언급 과대평가**: 가격, 예산, 통증 같은 단어의 짧은 언급을 실제 설명으로 취급한다.
4. **주제 전체와 개별 개념 혼동**: 유효한 특정 치료 근거가 주제 전체를 설명하는 것처럼 보일 수 있다.
5. **명시적 제외 범위 미반영**: 주제 명세의 제외 조건이 임베딩 순위에 반영되지 않는다.
6. **요약문 과대평가**: 글의 범위를 정확히 소개해도 실제 행동·기준·방법을 설명하지 않는 요약문은 직접 근거가 아니다.

## 사유 태그 집계

50건에 복수 선택된 사유 태그의 빈도다. 태그는 런타임 입력값이 아니라 재정렬 특성을 설계하고 결과를 평가하기 위한 사람의 판정 결과다.

| 태그 | 전체 | 관련 | 부분 관련 | 무관 | 해석 |
|---|---:|---:|---:|---:|---|
| `adjacent-topic` | 19 | 0 | 2 | 17 | 가장 큰 오탐 원인. 강한 감점 신호지만 하드 제외는 아님 |
| `required-concept` | 17 | 17 | 0 | 0 | 핵심 개념을 실제로 설명하는 강한 양성 신호 |
| `direct-answer` | 16 | 16 | 0 | 0 | 직접 답변 여부가 관련 판정과 가장 잘 일치 |
| `too-narrow` | 16 | 5 | 6 | 5 | 모든 라벨에 분포하므로 단독 감점 금지 |
| `supporting-concept` | 15 | 0 | 15 | 0 | 유효하지만 직접 근거보다 낮게 배치할 신호 |
| `brief-mention` | 9 | 0 | 7 | 2 | 요약문·도입부를 부분 근거로 낮추는 신호 |
| `wrong-surface-context` | 8 | 0 | 2 | 6 | 적용 위치·상황 불일치. 강한 감점 필요 |

| `keyword-collision` | 4 | 0 | 0 | 4 | 표현만 겹치는 오탐. 하드 제외에 가까운 신호 |
| `wrong-treatment` | 3 | 0 | 0 | 3 | 명확한 진료 불일치는 제외 가능 |

- `direct-answer + required-concept`: 16건, 모두 관련
- `brief-mention + supporting-concept`: 7건, 모두 부분 관련
- `supporting-concept + too-narrow`: 6건, 모두 부분 관련
- `adjacent-topic + incidental-mention`: 5건, 모두 무관
- `adjacent-topic + wrong-surface-context`: 5건, 모두 무관
- `adjacent-topic + wrong-treatment`: 3건, 모두 무관

## 재정렬 설계

아래는 현재 기준선을 바꾸지 않은 목표 설계다. 먼저 `retrieval-v2-shadow`로 실행하고 기준선보다 나아진 경우에만 기본 검색으로 승격한다.

### 설계 원칙

1. 임베딩 유사도 하나로 관련성·충족도·우선순위를 동시에 결정하지 않는다.
2. 검색은 주제 전체가 아니라 개념별로 수행한다.
3. 직접 근거와 보조 근거를 모두 보존하되 직접 근거를 먼저 배치한다.
4. 진료·환자·치아 위치·상황의 적용 범위를 별도 신호로 평가한다.
5. 명시적 제외와 확실한 문맥 불일치만 하드 제외하고, 좁은 범위와 환자군 차이는 감점 또는 범위 표시로 처리한다.
6. 검색 점수는 근거 후보를 찾는 데만 사용하며 개념 충족 판정은 별도 단계에서 수행한다.

### 1. 개념별 질의 생성

현재 구현은 주제 설명, 사용자 질문, 모든 개념과 기준을 한 질의에 합친다. 긴 질의에서 인접 의미가 섞이지 않도록 다음 세 종류의 표현을 분리한다.

- **주제 정체성 질의**: 어떤 진료·환자·치아 위치·상황에 관한 주제인지 확인
- **개념 질의**: 개별 `CoverageConcept`의 설명과 해당 기준만 포함
- **제외 질의**: `CoverageTopicSpec.exclusions`의 각 항목을 별도 부정 후보로 표현

개념 질의의 기본 형식은 다음과 같다.

```text
주제: {topic.label}
평가 개념: {concept.label}
개념 설명: {concept.description}
개념 식별 표현: {concept.retrievalHints.identityPhrases}
판정 기준: {criterion.label}: {criterion.description}
```

하나의 개념에 기준이 여러 개면 개념 벡터 하나와 기준별 벡터를 모두 만들 수 있다. 1차 구현은 비용을 줄이기 위해 개념 벡터로 후보를 모으고, 기준 문구는 규칙 검색 점수에 사용한다.

기존 `userQuestions`는 주제 전체 배열이라 개념과의 연결이 불분명하다. 다음 명세 버전에서는 개념에 `userQuestions` 또는 `queryExamples`를 두어 어떤 질문이 어느 개념을 찾는지 명시한다.

### 2. 후보 풀 구성

각 개념마다 다음 후보의 합집합을 만든다.

1. 개념 임베딩 상위 30건
2. 개념 식별 표현 규칙 검색 상위 20건
3. 주제 정체성 임베딩 상위 후보 중 개념 식별 표현이 일치한 문단

후보 수는 평가 단계에서 조정하되 처음부터 상위 10개로 자르지 않는다. 넓게 수집한 뒤 문맥과 직접성을 반영해 재정렬한다.

### 3. 공통 특성

| 특성 | 역할 | 초기 처리 방향 |
|---|---|---|
| 개념 임베딩 유사도 | 해당 개념과 의미적으로 가까운지 | 가장 큰 기본 점수 |
| 주제 정체성 유사도 | 진료·환자·위치·상황이 맞는지 | 두 번째 기본 점수 |
| 기준 규칙 일치 | 실제 판정 문구가 있는지 | 직접성 가산점 |
| 개념 식별 표현 | 명시적 핵심 표현이 있는지 | 후보 수용의 필수 관문 |
| 구조화된 범위 일치 | 진료·환자군·치아 위치·상황 | 일치 가산, 명확한 불일치 감점 |
| 제외 질의 유사도 | 부분교정 등 명시적 제외에 가까운지 | 강한 감점 또는 고신뢰 시 제외 |
| 근거 종류 | 요약·도입·상세 섹션·FAQ 여부 | 설명 없는 요약문 감점 |
| 문서·개념 중복 | 같은 글의 유사 문단 반복 여부 | 다양성 제약 |

초기 결합은 원시 코사인 값의 단순 합보다 주제별 순위 백분위 또는 Reciprocal Rank Fusion을 사용한다. 질의마다 코사인 분포가 달라 동일 임계값을 바로 적용하기 어렵기 때문이다.

현재 shadow 가중치는 `개념 의미 30% + 개념 식별 30% + 주제 정체성 15% + 기준 규칙 10% + 범위 일치 10% + 근거 구체성 5%`다. 단, 가중합 전에 개념 식별 표현과 필수 문맥을 통과해야 하며 명시적 제외 표현은 후보에서 제거한다. 이 수치는 확정값이 아니라 현재 라벨로 비교할 shadow 설정이다.

### 4. 감점과 제외 정책

- `wrong-treatment`, `keyword-collision`, `incidental-mention`에 대응하는 명확한 구조 신호는 강한 감점 또는 제외한다.
- `adjacent-topic`, `wrong-surface-context`는 강하게 감점하되 일부 부분 근거가 있으므로 자동 제외하지 않는다.
- `brief-mention`에 해당하는 요약문은 후보로 유지하지만 상세 섹션보다 낮게 배치한다.
- `too-narrow`는 관련 근거에도 5건 존재하므로 점수 자체를 낮추지 않는다. 대신 적용 범위를 결과에 기록하고 다른 개념의 근거로 확장하지 않는다.
- `wrong-patient`는 현재 유일한 사례가 부분 관련이므로 환자군이 다르다는 이유만으로 제외하지 않는다.
- 같은 문서에서 요약문과 상세 문단이 함께 검색되면, 상세 문단이 기준을 직접 설명하는 경우 상세 문단을 우선한다.

### 5. 다양성 선택

개념별 최종 후보는 상위 5건을 기본으로 하며 같은 문서에서는 개념당 1개 문단을 우선한다. 주제 화면의 통합 상위 10건은 다음 원칙으로 구성한다.

1. 필수 개념마다 사용할 수 있는 후보가 있으면 최소 1건 확보
2. 직접 근거를 보조 근거보다 먼저 배치
3. 같은 문서 최대 2건 유지
4. 같은 문서의 요약과 상세 문단이 경쟁하면 상세 문단 우선
5. 추천 개념은 필수 개념 후보가 확보된 뒤 남은 슬롯에 배치

후보가 없는 개념은 슬롯을 억지로 채우지 않는다. 다양성 선택을 마친 뒤 통합 후보는 `rerankScore` 순으로 다시 정렬한다.

### 6. 검색 결과 계약

재정렬 결과는 단일 점수만 노출하지 않고 다음 정보를 보존한다.

```typescript
interface ConceptEvidenceCandidate {
  topicSpecId: string;
  conceptId: string;
  evidenceUnitId: string;
  semanticRank: number;
  lexicalRank: number | null;
  topicScopeScore: number;
  conceptScore: number;
  conceptIdentityScore: number;
  rerankScore: number;
  matchedCriteria: string[];
  identityMatches: string[];
  contextMatches: string[];
  scopeWarnings: string[];
  exclusionMatches: string[];
  evidenceLevel: "direct" | "supporting" | "discovery-only";
  reasons: string[];
}
```

`discovery-only`는 관련 글을 찾는 데는 유용하지만 개념 충족 근거로 단독 사용하면 안 되는 요약문과 도입부를 뜻한다.

## 개념별 검색 평가 방식

현재 라벨은 `(주제, 근거 문단)`의 관련성만 평가한다. 개념별 검색을 검증하려면 새 후보에 대해 어떤 개념을 실제로 지원하는지도 기록해야 한다.

- 기존 50건의 주제 라벨은 재정렬 정밀도 비교에 그대로 재사용한다.
- 동일 `evidenceUnitId`는 기존 라벨과 메모를 자동 연결한다.
- 새로 등장한 후보만 `관련/부분 관련/무관`을 추가 검토한다.
- 개념별 상위 후보에는 `supports/partial/not-supported` 개념 라벨을 추가한다.
- 임상 개념은 사전 라벨 후 의료진 확인 상태를 별도로 둔다.

평가 지표는 다음과 같이 확장한다.

| 지표 | 목적 |
|---|---|
| 주제별 usable precision@3 | 상위 결과가 실제 근거인지 |
| 주제별 irrelevant rate@10 | 오탐이 얼마나 남았는지 |
| 필수 개념 hit rate@5 | 각 필수 개념에 사용 가능한 후보가 있는지 |
| 개념별 usable precision@3 | 개념 질의가 해당 개념 근거를 찾는지 |
| direct evidence rate | 보조·요약이 아니라 직접 근거가 얼마나 상위에 있는지 |
| exclusion leakage rate | 명시적 제외 후보가 상위 결과에 남는지 |
| new-review count | 새 방식 도입에 필요한 추가 라벨링 양 |

승격 조건은 기존 두 품질 목표를 유지하면서, 필수 개념 hit rate가 기존보다 낮아지지 않고 exclusion leakage가 감소하는 것이다. 어떤 개념에서도 사용할 후보가 없으면 이를 곧바로 콘텐츠 공백으로 확정하지 않고 `근거 검색 불충분` 상태로 남긴다.

## 구현 순서

1. 현재 50건 라벨과 사유 태그 집계를 평가 출력에 포함 — 완료
2. 개념별 질의와 후보 합집합을 만드는 shadow 검색 구현 — 완료
3. 요약문 우선순위와 명시적 제외 처리 — 완료
4. 구조화된 범위 정보가 없는 동안 카테고리·제목·경로 기반 범위 특성 적용 — 완료
5. 기준선과 shadow 결과 비교 리포트 생성 — 완료
6. 새 후보와 개념 라벨을 관리자 검토 화면에서 추가 입력 — 구현 완료
7. 결과를 보고 구조화된 `EvidenceScope` 도입 여부 결정

## 다음 단계

`retrieval-v2-shadow.3`은 신규 후보 직접 검토 결과를 반영한다. 요약문을 통합 근거에서 제외하고, 개념 식별 표현이 본문에 명시된 경우만 수용하며, 문서 전체의 제외 표현을 모든 문단에 적용한다. `색`, `칫솔`처럼 넓은 단어는 제거하고 `색조·투명도`, `양치법·45도 각도`처럼 판정 가능한 표현으로 좁혔다. 기준 일치 계산에서는 개념명을 제외해 하나의 개념명이 모든 기준을 동시에 충족시키는 누수를 막았다.

직접 검토에서 확인한 다음 오류는 리포트 생성 시 회귀 조건으로 검증한다.

- 투명교정 요약문을 정상 통증 근거로 사용하지 않음
- 성인 흔들림 콘텐츠를 소아 외상 근거로 사용하지 않음
- 소아 앞니 문단은 `diagnosis` 보조 근거로만 연결하고 `aesthetics`에서 제외
- 치간도구·구강청결제 사용법을 `brushing`으로 분류하지 않음
- 구강청결제 콘텐츠를 `devices` 후보로 연결

후보를 억지로 채우지 않은 결과 다음 개념은 현재 검색 결과가 비어 있다.

- 교정: `root-resorption`, `gum-recession`
- 비용·보험: `private-insurance`
- 앞니 치료: `diagnosis`
- 소아 외상: `treatment-followup`

다음 단계는 새 후보만 주제 관련성으로 검토하고, 각 개념 후보에 `supports/partial/not-supported` 라벨을 추가하는 것이다. 그 결과로 식별 표현과 수용 임계값을 조정한 뒤 구조화된 `EvidenceScope` 도입 여부를 결정한다.

관리자 화면 `/admin/content/strategy/concept-review`에서 통합 후보 28건과 개념 판정 61건을 검토한다. 기존 기준선에 포함된 13건은 주제 라벨을 자동 연결하고, 신규 15건은 주제 라벨부터 입력한다. 저장 단위는 후보 하나이며 해당 후보에 연결된 모든 개념을 판정해야 완료로 집계한다.

소아 치아 외상과 앞니 치료처럼 임상 판단이 포함되는 주제는 사전 라벨과 별도로 의료진 최종 확인이 필요하다.
