# /admin UX Verification Checklist

> 목적: `/admin`을 기본 관리자 콘솔로 운영할 때 핵심 사용자 흐름이 깨지지 않았는지 빠르게 점검하기 위한 체크리스트.

## 1. 기본 진입 / 리디렉션

- [ ] `/admin` 접속 시 로그인 전에는 `/admin/login`으로, 로그인 후에는 관리자 콘솔로 진입한다.
- [ ] `/admin/preview/[category]/[slug]`에서 초안 미리보기가 정상 동작한다.

## 2. 공개 페이지의 관리자 진입점

- [ ] 일반 페이지 우하단 관리자 버튼이 `/admin`으로 이동한다.
- [ ] 블로그 페이지에서 관리자 버튼 클릭 시 `/admin?tab=content&sub=posts`로 이동한다.
- [ ] Footer/섹션의 설정 편집 링크가 `/admin`을 연다.
- [ ] 블로그 상세의 초안 관리 바/인라인 편집 버튼이 `/admin`으로 이동한다.

## 3. 로그인 / 인증

- [ ] `/admin/login`에서 Google 로그인 버튼이 정상 동작한다.
- [ ] 허용된 관리자 계정 로그인 시 `/admin`으로 진입한다.
- [ ] 허용되지 않은 계정 로그인 시 차단 메시지가 보인다.
- [ ] 로그아웃 후 `/admin/login`으로 돌아간다.

## 4. 대시보드 탭

- [ ] 기본 탭이 `대시보드`로 열린다.
- [ ] 대시보드 상단의 CTA(예약 발행 보기, 콘텐츠 관리 열기)가 정상 동작한다.
- [ ] 요약 카드(세션/검색 클릭/검색 노출/CTA/전화 클릭)가 로딩 또는 데이터를 정상 표시한다.
- [ ] “지금 할 일” 카드 클릭 시 관련 탭으로 이동한다.

## 5. 콘텐츠 탭

- [ ] `콘텐츠 > 포스트 관리`에서 포스트 목록이 보인다.
- [ ] `콘텐츠 > 포스트 관리`에서 검색/필터/정렬이 동작한다.
- [ ] 블로그 편집 버튼으로 편집기가 열린다.
- [ ] 새 포스트 작성이 열린다.
- [ ] 발행/예약 발행이 동작한다.
- [ ] `콘텐츠 > 발행 일정`에서 발행 요일 저장이 동작한다.
- [ ] `콘텐츠 > 성과`에서 카테고리 분포/인기 포스트가 보인다.

## 6. 유입·SEO 탭

- [ ] `트래픽` 서브탭 데이터가 보인다.
- [ ] `검색 성과`에서 수정/관련 글 작성 버튼 클릭 시 `/admin` 내부의 콘텐츠 탭으로 이동한다.
- [ ] `콘텐츠 전략`에서 새 글 작성 CTA 클릭 시 `/admin` 내부로 이동한다.
- [ ] `트렌드` 상세 카드와 기간 전환이 동작한다.

## 7. 전환 탭

- [ ] 전환 탭이 열리고 PostHog 설정 여부에 따라 안내 또는 데이터가 보인다.
- [ ] 기간 전환이 동작한다.
- [ ] CTA 위치 / 전환 페이지 / 블로그 기여 표가 정상 렌더링된다.

## 8. 사이트 설정 탭

- [ ] SNS 링크 저장이 동작한다.
- [ ] 병원 정보 저장이 동작한다.
- [ ] 진료시간 저장이 동작한다.
- [ ] 저장 후 공개 페이지(Footer/Contact/About)에 반영된다.

## 9. 개발도구 탭

- [ ] 개발도구 탭은 기본 진입이 아니고, 필요할 때만 열린다.
- [ ] 현황 / 성능 / 레퍼런스 / 모니터링 서브탭이 기존과 동일하게 동작한다.

## 10. 회귀 체크

- [ ] `/admin` 운영 중에도 공개 페이지 GA 비활성화 조건이 관리자 브라우저에서 유지된다.
- [ ] `/admin2*` 경로는 더 이상 운영 표면이 아니며, 외부 링크/문서에서 사용하지 않는다.
- [ ] 빌드 시 `/admin` 관련 라우트만 정상 생성된다.
- [ ] SEO/블로그/설정 핵심 흐름이 모두 `pnpm build` 기준으로 통과한다.
