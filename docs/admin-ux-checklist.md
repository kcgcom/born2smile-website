# /admin UX Verification Checklist

> 목적: `/admin` 운영 콘솔의 핵심 흐름이 깨지지 않았는지 빠르게 회귀 점검하기 위한 체크리스트.

## 1. 기본 진입 / 리디렉션

- [ ] 로그인 전 `/admin` 접속 시 `/admin/login`으로 이동한다.
- [ ] 로그인 후 `/admin` 접속 시 기본 탭 `?tab=dashboard` 흐름으로 진입한다.
- [ ] `/admin/preview/[category]/[slug]` 초안 미리보기가 정상 동작한다.
- [ ] 레거시 쿼리(`?tab=blog`, `?tab=traffic`, `?tab=search`, `?tab=trend`, `?tab=dev`)가 새 탭 구조로 리디렉션된다.

## 2. 공개 페이지의 관리자 진입점

- [ ] 일반 페이지 우하단 관리자 버튼이 `/admin`으로 이동한다.
- [ ] 블로그 페이지의 관리자 버튼이 `/admin?tab=content&sub=posts`로 이동한다.
- [ ] Footer/섹션의 설정 편집 링크가 `/admin?tab=settings`로 이동한다.
- [ ] 블로그 상세의 초안 관리 바/인라인 편집 버튼이 올바른 편집 딥링크를 연다.

## 3. 로그인 / 인증

- [ ] `/admin/login`의 Google 로그인 버튼이 정상 동작한다.
- [ ] 허용된 관리자 계정 로그인 시 콘솔로 진입한다.
- [ ] 허용되지 않은 계정 로그인 시 차단 메시지가 보인다.
- [ ] 로그아웃 후 `/admin/login`으로 돌아간다.

## 4. 대시보드 탭 (`tab=dashboard`)

- [ ] 상단 CTA `예약 발행 보기`, `콘텐츠 관리 열기`가 동작한다.
- [ ] 요약 카드(주간 세션/검색 클릭/검색 노출/총 CTA 클릭/전화 클릭)가 로딩 또는 데이터를 정상 표시한다.
- [ ] `지금 할 일` 카드 클릭 시 관련 탭/서브탭으로 이동한다.

## 5. 콘텐츠 탭 (`tab=content`)

- [ ] `sub=posts`에서 포스트 목록, 검색, 필터, 정렬이 동작한다.
- [ ] 편집/새 포스트 작성이 `BlogEditor`로 열린다.
- [ ] 발행 팝업에서 `스케줄`, `즉시 발행`, `날짜 직접 선택` 세 모드가 동작한다.
- [ ] `sub=schedule`에서 발행 요일 저장이 동작한다.
- [ ] `sub=stats`에서 카테고리 분포/인기 포스트가 렌더링된다.

## 6. 유입·SEO 탭 (`tab=seo`)

- [ ] `sub=traffic` 데이터가 정상 표시된다.
- [ ] `sub=search`의 수정/관련 글 작성 CTA가 콘텐츠 탭으로 연결된다.
- [ ] `sub=strategy`의 새 글 작성 CTA가 콘텐츠 탭으로 연결된다.
- [ ] `sub=trend`의 기간 전환과 상세 카드가 동작한다.

## 7. 전환 탭 (`tab=conversion`)

- [ ] PostHog 미설정 시 안내 메시지가, 설정 시 데이터 카드/표가 보인다.
- [ ] 기간 전환(7d/30d/90d)이 동작한다.
- [ ] CTA 위치 / 전환 페이지 / 블로그 기여 표가 정상 렌더링된다.

## 8. 사이트 설정 탭 (`tab=settings`)

- [ ] SNS 링크 저장이 동작한다.
- [ ] 병원 정보 저장이 동작한다.
- [ ] 진료시간 저장이 동작한다.
- [ ] 저장 후 공개 페이지(Footer/Contact/About)에 반영된다.

## 9. 개발도구 탭 (`tab=devtools`)

- [ ] `sub=project`, `sub=perf`, `sub=ref`, `sub=monitoring`, `sub=ai`가 모두 열린다.
- [ ] 모니터링 탭의 Sentry/PostHog 상태 카드가 렌더링된다.
- [ ] AI 로그 탭에서 최근 AI 작성 로그를 확인할 수 있다.

## 10. 회귀 체크

- [ ] 관리자 로그인 브라우저에서는 공개 페이지 GA 추적 비활성화 조건이 유지된다.
- [ ] `/admin2*` 경로는 더 이상 운영 표면이 아니며 외부 링크/문서에서 사용하지 않는다.
- [ ] 빌드 시 `/admin` 관련 라우트만 정상 생성된다.
- [ ] 핵심 운영 흐름이 `pnpm build` 기준으로 통과한다.
