# 키워드 후보 추천 v2

## 목표

네이버 SearchAd 연관 키워드를 저장 단계에서 버리지 않고, 택소노미 핵심어·콘텐츠·FAQ·제품·지역 검색으로 목적을 나눈 뒤 검증된 추천만 관리자 큐에 노출한다.

## 현재 기준선

최신 스냅샷 기준 전체 연관 키워드 7,237개 중 문자열 35% 매칭 746개, 월 검색량 100회 이상 411개, 서브그룹별 상위 5개 148개가 현재 추천 대상이 된다. 문자열 매칭과 상위 5개 제한이 주요 누락 지점이다.

## 원칙

1. 원본 연관 키워드는 모두 보존한다.
2. 택소노미 핵심어와 콘텐츠 기회를 별도 용도로 분류한다.
3. 하드 필터는 명백한 비치과 노이즈에만 사용한다.
4. 문자열·의미 유사도·검색 의도·검색량·반복 관측을 별도 신호로 보존한다.
5. 운영 교체 전 현행 알고리즘과 그림자 모드를 동일한 평가 세트로 비교한다.

## 실행 단계

### 1. 평가 세트

전체 풀에서 300개를 자동 층화 추출한다. 현재 노출, 상위 5개 제한 탈락, 문자열 매칭 탈락, 저검색량, 제품·브랜드, 지역, 노이즈를 모두 포함한다. 시스템이 초벌 라벨을 만들고 사람은 확인하거나 수정한다.

`pnpm exec tsx scripts/generate-keyword-candidate-evaluation.ts`가 활성 SearchAd 스냅샷을 읽어 `.tmp/keyword-candidate-evaluation.json`을 만든다. 이 파일은 운영 DB를 변경하지 않으며, 초벌 라벨은 정답이 아닌 검수 보조 정보로 취급한다.

### 2. 검수 UI

관리자는 관련성, 추천 목적, 카테고리·서브그룹, 최종 판단만 검수한다. 모델 동의·고신뢰 항목은 일괄 확인할 수 있어야 한다.

### 3. 그림자 추천

문자열 검색과 Gemini 임베딩 검색 결과를 합치고 복수 서브그룹 점수를 보존한다. 검색량은 로그 변환과 카테고리 백분위를 사용하며 제품·지역·모호성은 별도 위험 신호로 표시한다.

`pnpm exec tsx scripts/evaluate-keyword-candidate-shadow.ts`는 확정된 300개를 leave-one-out 방식으로 평가해 현행 방식과 그림자 추천을 비교한다. 이 점수는 모델·규칙 보정에 사용한 평가 세트의 교차검증 결과이므로 독립 검증 점수로 간주하지 않는다.

`pnpm exec tsx scripts/run-keyword-candidate-shadow.ts`는 평가 세트와 동일한 SearchAd 스냅샷의 전체 연관 키워드에 그림자 추천을 실행하고 `.tmp/keyword-candidate-shadow-full.json`을 만든다. 운영 후보와 택소노미는 변경하지 않는다. 이미 라벨링한 300개는 학습 참조로 사용하되 독립 감사 큐에서는 제외하고, 목적별 상위 20개씩 최대 100개를 별도 `auditQueue`로 보존한다. `--publish` 옵션을 붙이면 이 감사 큐만 관리자 검수용 Supabase 캐시에 게시한다.

독립 감사 화면(`/admin/content/trends/keyword-evaluation/shadow-audit`)은 사람의 최초 판단을 저장하기 전까지 모델 예측을 숨긴다. 저장 후에는 예상 목적·조치·분류 후보를 공개해 불일치 사례를 확인할 수 있다.

### 4. 전환 기준

- 유효 키워드 재현율 90% 이상
- 각 목적 큐 상위 20개 정밀도 70% 이상
- 평가 세트에서 주요 검색 의도 누락 0건
- 한 번의 기본 검토량 100개 이하
- 원본 저장 단계의 조용한 탈락 0건

## 롤백

v2는 그림자 모드에서만 실행한다. 전환 전까지 기존 `refreshKeywordTaxonomyCandidates()`가 운영 결과를 계속 제공하며, v2 결과는 별도 평가 데이터로만 저장한다.

전체 풀에서 추천 수가 많다는 이유만으로 모두 운영 큐에 노출하지 않는다. 독립 감사 큐 100개의 목적별 정밀도를 먼저 확인하고, 기준을 통과한 이후에만 신뢰도 임계값과 운영 큐 상한을 확정한다.

`keyword-shadow-v2`는 확정된 초기 평가 300개와 1차 독립 감사 100개를 함께 참조한다. 모델의 관련 판정과 관리자 노출을 분리하고, 치과 용어 근거·의미 유사도·명백한 타 진료과 신호를 조합한 검토 노출 게이트를 적용한다. 새 엔진 감사 큐를 게시할 때 이전 감사 세트와 라벨은 `api_cache`의 history 키로 보관한다.

`keyword-shadow-v3`는 누적 확정 라벨의 분류 위치 투표와 검색 의도 신호를 의미·문자열 점수에 결합해 택소노미 후보를 재정렬한다. 비용, 교정 방법, 심미 고민, 소아 교정, 병원 추천, 디지털 임플란트, 구강위생, 시림, 보철 소재 등 재사용 가능한 의도만 명시적으로 반영한다.

`keyword-shadow-v4`는 3차 감사 실패 패턴을 일반화한다. 부정교합 유형은 교정 방법이 아니라 심미/고민으로 우선 배치하고, 라미네이트처럼 명시적인 시술명은 동일 서브그룹을 우선한다. `CROWN`처럼 치과와 비치과 의도가 혼재한 단독 영문어는 자동 추천하지 않고 검토로 보내며, Toyota Crown과 눈매교정처럼 치과 용어와 표면적으로 겹치는 타 도메인 검색어는 제외한다.

`keyword-shadow-v5`는 4차 감사에서 확인된 목적 큐 혼동을 보강한다. 병원·전문의 탐색은 지역/병원 목적, 제품명은 제품 목적, 넓은 질환 주제는 택소노미 목적, 증상형 표현은 FAQ 목적으로 분리한다. 교정 비용과 연령대 의도도 각각 비용/기간과 대상/나이 서브그룹을 우선한다.

## 라벨링 종료 구조

v5 감사 100개는 최종 홀드아웃으로 고정하고 이후 추천 학습 참조에서 제외한다. 기존 초기 평가와 v1~v4 감사 라벨만 학습·교차검증에 사용한다. 홀드아웃은 목적별 정밀도와 택소노미 정확도를 한 번 판단하는 용도이며, 결과를 보고 같은 세트를 다시 보정하지 않는다.

추가 검토는 엔진 버전마다 100개를 반복하지 않는다. 관련성·목적 신뢰도가 낮거나 제품/지역 신호와 목적이 충돌하거나 택소노미 상위 후보 간 점수 차가 작은 사례만 `boundaryReviewQueue`에 최대 30개까지 담는다. 이 큐는 자동 승인·자동 제외에 사용하지 않고 관리자 판단이 필요한 경계 사례를 찾는 용도로만 사용한다. `--publish-boundary`로 게시한 큐는 `/admin/content/trends/keyword-evaluation/boundary-review`에서 검토한다.

## 제외 범위

- 평가 전에 기존 189개 후보 상태를 자동 변경하지 않는다.
- 평가 기준을 통과하기 전에 운영 택소노미를 자동 수정하지 않는다.
- 제품·브랜드 키워드를 일괄 제외하지 않는다.
