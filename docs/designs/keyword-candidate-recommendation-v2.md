# 키워드 후보 추천 v2

## 목표

네이버 SearchAd 연관 키워드를 저장 단계에서 버리지 않고, 택소노미 핵심어·콘텐츠·FAQ·제품·지역 검색으로 목적을 나눈 뒤 검증된 추천만 관리자 큐에 노출한다.

## 현재 기준선

최신 스냅샷 기준 전체 연관 키워드 7,237개 중 문자열 35% 매칭 746개, 월 검색량 100회 이상 411개, 서브그룹별 상위 5개 148개가 현재 추천 대상이 된다. 문자열 매칭과 상위 5개 제한이 주요 누락 지점이다.

## 원칙

1. 원본 연관 키워드는 모두 보존한다.
2. 택소노미 핵심어와 콘텐츠 기회를 별도 용도로 분류한다.
3. 하드 필터는 명백한 비치과 노이즈에만 사용한다.
4. 문자열·의미 유사도·검색 의도·검색량·반복 관측을 별도 신호로 보존한다.
5. 운영 교체 전 현행 알고리즘과 그림자 모드를 동일한 평가 세트로 비교한다.

## 실행 단계

### 1. 평가 세트

전체 풀에서 300개를 자동 층화 추출한다. 현재 노출, 상위 5개 제한 탈락, 문자열 매칭 탈락, 저검색량, 제품·브랜드, 지역, 노이즈를 모두 포함한다. 시스템이 초벌 라벨을 만들고 사람은 확인하거나 수정한다.

`pnpm exec tsx scripts/generate-keyword-candidate-evaluation.ts`가 활성 SearchAd 스냅샷을 읽어 `.tmp/keyword-candidate-evaluation.json`을 만든다. 이 파일은 운영 DB를 변경하지 않으며, 초벌 라벨은 정답이 아닌 검수 보조 정보로 취급한다.

### 2. 검수 UI

관리자는 관련성, 추천 목적, 카테고리·서브그룹, 최종 판단만 검수한다. 모델 동의·고신뢰 항목은 일괄 확인할 수 있어야 한다.

### 3. 그림자 추천

문자열 검색과 Gemini 임베딩 검색 결과를 합치고 복수 서브그룹 점수를 보존한다. 검색량은 로그 변환과 카테고리 백분위를 사용하며 제품·지역·모호성은 별도 위험 신호로 표시한다.

### 4. 전환 기준

- 유효 키워드 재현율 90% 이상
- 각 목적 큐 상위 20개 정밀도 70% 이상
- 평가 세트에서 주요 검색 의도 누락 0건
- 한 번의 기본 검토량 100개 이하
- 원본 저장 단계의 조용한 탈락 0건

## 롤백

v2는 그림자 모드에서만 실행한다. 전환 전까지 기존 `refreshKeywordTaxonomyCandidates()`가 운영 결과를 계속 제공하며, v2 결과는 별도 평가 데이터로만 저장한다.

## 제외 범위

- 평가 전에 기존 189개 후보 상태를 자동 변경하지 않는다.
- 평가 기준을 통과하기 전에 운영 택소노미를 자동 수정하지 않는다.
- 제품·브랜드 키워드를 일괄 제외하지 않는다.
